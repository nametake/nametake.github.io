先日 protoc-gen-gohttp で使うProtocol BuffersのAPIをV1からV2に移行しました。

その際、V2 APIでプラグインを生成するにあたっていくつか知見が溜まったのでメモとして残そうと思います。

エントリポイント Link to heading

V1の頃は標準入力と標準出力を自分で制御する必要がありましたが、 protogen.Option を使うことでそのあたりをいい感じにやってくれるようになりました。

1

protogen.Options{}.Run(func(p *protogen.Plugin) error {})

Run に渡すCallbackの protogen.Plugin を使うことで内部的に標準入力と標準入力をいい感じにやってくれます。

ファイル生成 Link to heading

protogen.Plugin が実装している NewGeneratedFile を使用するとファイル対応するファイルが生成されます。

1
2

7

for _, file := range p.Files {
	if file.Generate {
		filename := file.GeneratedFilenamePrefix + ".http.go"
		g := p.NewGeneratedFile(filename, file.GoImportPath)
		// TODO
	}
}

上記例では protogen.Plugin が持っている File のリストから情報を取り出すようにしているためProtoファイルごとにコードの生成が生成されます。

コードの出力 Link to heading

GeneratedFile に実装されている P メソッドを使うと生成したファイルに文字を書き込むことができます。

1

g.P("// Code generated by protoc-gen-gohttp. DO NOT EDIT.")

ドキュメントに書いてあるとおり、 fmt.Print と同じルールで書き込まれるため以下のように変数を渡すことでその値を出力できます。

1

g.P("// source: ", file.Desc.Path())
